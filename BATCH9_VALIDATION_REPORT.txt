ROOT CAUSE — INFRA_HASH É PER-AÇÃO, NÃO PER-SESSÃO
═══════════════════════════════════════════════════════════════

O código em _view_document_html_core (linhas 3730-3742) faz:
  1. Extrai um infra_hash qualquer da árvore HTML
  2. Constrói URL: documento_imprimir_web&id_documento=X&infra_hash=Y

Mas o SEI responde com redirect ao login:
  "Hash inválido. [/sei/controlador.php?acao=documento_imprimir_web
   &id_documento=42248674&infra_hash=1f16088b...]"

O infra_hash é VINCULADO À AÇÃO que o gerou. Um hash gerado para
procedimento_visualizar não funciona para documento_imprimir_web.
Não é possível construir URLs com hashes emprestados.

Code sketch:

  def view_document_html(self, id_documento, id_procedimento):
      arvore_html = self._navigate_to_arvore(id_procedimento)
      if not arvore_html:
          raise RuntimeError(...)
      
      with self._auto_unit_switch(arvore_html) as switched_to:
          if switched_to:
              arvore_html = self._navigate_to_arvore(id_procedimento)
          
          try:
              return self._view_document_html_core(id_documento, arvore_html)
          except RuntimeError:
              # Fallback: expand all folders, use TreeDocument.src_url
              tree = self.get_full_document_tree(id_procedimento, expand_all=True)
              target = [d for d in tree if d.id_documento == str(id_documento)]
              if not target or not target[0].src_url:
                  raise
              r = self._get(target[0].src_url)
              if "login.php" in str(r.url) or len(r.text) < 500:
                  raise RuntimeError("Failed to fetch document via src_url")
              return r.text
