## Bug Report & Enhancement: `sign_block` — Encoding + Fluxo Canônico + Preview

### Problema Encontrado

`sign_block('774681')` reportou falha mas **a assinatura foi efetivada** no servidor (confirmado às 20:56). O erro é **falso negativo** na detecção de sucesso.

### Root Cause: Encoding Latin-1

O SEI usa `charset=ISO-8859-1`. O campo `selCargoFuncao` contém `2º Tenente QOEM BM` onde `º` = `\xBA` em Latin-1.

**O que acontece:**
1. `_sign_from_blocos` faz POST intermediário pra chegar ao form de assinatura
2. httpx decodifica a resposta HTML como UTF-8 por padrão (mesmo com header `charset=iso-8859-1`)
3. BeautifulSoup lê `\xBA` como codepoint inválido, `select.find("option", selected=True)` retorna `None`
4. O fallback pega o primeiro option não-null, mas o valor já está corrompido na string Python
5. Na hora do `urlencode(..., encoding='iso-8859-1')`, o `º` (que veio via UTF-8 decode como `\xC2\xBA`) → SEI recebe `2Âº` → rejeita: "Cargo/Função não está associado com este usuário"
6. Porém, **o servidor processou a assinatura antes** de retornar o formulário de erro — falso negativo

**Fix de encoding:** Ao parsear qualquer resposta do SEI, forçar `response.content.decode('latin-1')` antes de passar pro BeautifulSoup, ou setar `response.encoding = 'iso-8859-1'` antes de ler `.text`. Isso vale pra **todo o client**, não só sign.

### Fluxo Atual vs Fluxo Real da UI

**Fluxo atual (`_sign_from_blocos`):**
1. GET `/bloco_assinatura_listar`
2. Extrai URL `documento_assinar` via regex no JS
3. POST com `hdnInfraItemId=<bloco>` → recebe `frmAssinaturas`
4. Preenche senha + sobrescreve cargo/órgão com credentials → POST → resultado

**Fluxo real da UI:**
1. Na listagem de blocos, clicar no número do bloco → `rel_bloco_protocolo_listar&id_bloco=XXX`
2. Página "Documentos do Bloco de Assinatura XXXXX" com tabela: Seq, Processo, Documento, Tipo, Assinaturas, Ações
3. **Preview:** Clicar no número SEI do documento abre um `divInfraSparklingModal` com o conteúdo renderizado do documento — permite checar do que se trata sem navegar pro processo/árvore
4. Marcar checkboxes dos docs desejados (assinatura em lote)
5. Clicar "Assinar" → abre `frmAssinaturas` **já pré-preenchido** (Órgão, Assinante, Cargo)
6. Inserir senha → submeter
