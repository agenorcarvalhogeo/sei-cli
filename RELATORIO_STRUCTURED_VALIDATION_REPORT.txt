FIX NEEDED — TWO PARTS
═══════════════════════════════════════════════════════════════

PART 1: Make view_document_html work for cross-folder documents

The fix: view_document_html() should expand all folders before searching
for the print/view URL. The tree already has get_full_document_tree() with
expand_all=True that does this.

Option A (preferred — minimal change):
  In view_document_html(), after _navigate_to_arvore(), if the document
  is not found in the initial HTML, expand all folders using the same
  AJAX pattern that get_full_document_tree uses, then re-search.

Option B (simpler — construct URL directly):
  The documento_imprimir_web URL format is:
    controlador.php?acao=documento_imprimir_web&id_documento=XXXXX&infra_hash=YYYYY
  
  The infra_hash is per-page (not per-document) and is already available
  from the arvore HTML even without expanding folders. So we can:
    1. Extract infra_hash from arvore_html (it's in many links already)
    2. Construct the URL: f"controlador.php?acao=documento_imprimir_web&id_documento={id_documento}&infra_hash={infra_hash}"
    3. GET that URL and return the HTML
  
  This avoids expanding folders entirely. The infra_hash is already in
  _view_document_html_core's search context — we just need a fallback
  that constructs the URL instead of only searching for it.

  Code sketch for Option B (in _view_document_html_core):
  
    # After both regex searches fail:
    # Fallback: construct URL directly using infra_hash from any link in arvore
    hash_match = re.search(r'infra_hash=([a-f0-9]+)', arvore_html)
    if hash_match:
        constructed_url = (
            f"controlador.php?acao=documento_imprimir_web"
            f"&id_documento={id_documento}"
            f"&infra_hash={hash_match.group(1)}"
        )
        r = self._get(self._sei_url(constructed_url))
        if "login.php" not in str(r.url) and len(r.text) > 500:
            return r.text
    
    raise RuntimeError(f"Link de visualização não encontrado para documento {id_documento}")

                            Best quality. Works for unsigned internal docs
                            that the user can edit.

                            ONLY acceptable for true binary PDFs.
