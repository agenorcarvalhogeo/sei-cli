🔴 BUG CRÍTICO: "Link de visualização não encontrado"
================================================================================

AFETA: docs internos de pastas que não são a última expandida + todos os PDFs

CAUSA RAIZ (confirmada por testes manuais com a API Python):

  O process-read usa view_document(id_doc, id_proc) que busca um link de
  visualização HTML na página. Esse link só existe:
    - Para docs INTERNOS que estão na pasta ATUALMENTE ATIVA na sessão SEI
    - NUNCA para docs tipo 'pdf' (PDFs externos não têm visualização HTML)

  Quando o código navega entre pastas, o estado da sessão muda e os links
  de visualização das pastas anteriores somem.

  Teste 3: PASTA4 ✅ (3/3), PASTA3 ❌ (3/3)
  Teste 4: PASTA2 ✅ (2/2), PASTA1 ❌ (3/3)

   Correto semanticamente (o envelope é ok=true), mas pode confundir pipes.

          # read_document faz goto interno pelo id

      elif doc.tipo == 'pdf':
          # Baixar bytes via src_url do TreeDocument

POR QUE RESOLVE:

  1. get_full_document_tree(expand_all=True) expande tudo de uma vez
     → estado de sessão consistente
  2. Cada TreeDocument tem src_url IMUTÁVEL populado no momento da expansão
  3. PDFs usam download_document() que já funciona (testado: 105KB OK)
  4. Internos usam read_document(id_doc, id_proc) — navega por ID fixo
