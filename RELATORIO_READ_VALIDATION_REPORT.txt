CRITICAL IMPROVEMENT: USE HTML READING, NOT PDF TEXT FALLBACK
═══════════════════════════════════════════════════════════════

The current relatorio-read flow for non-editor documents:
  1. Try client.read_relatorio() → get_editor_sections() → parse_relatorio()
  2. If that fails → _document_read_core() → plain text → _fallback_relatorio_from_text()

This loses ALL structural information. The fix:

PREFERRED APPROACH — read document HTML via goto/iframe navigation:

The SEI iframe displays document content as rendered HTML even for
documents that can't be opened in the editor. The goto approach
(read_document_content_retry / direct-document-id navigation) already
navigates to the document and opens its iframe. Instead of extracting
plain text from the response, we should extract the HTML body and pass
it to parse_relatorio().

Relevant existing functions that return HTML:

  client.view_document_html(id_documento, id_procedimento)
    → Uses documento_imprimir_web → returns full rendered HTML
    → Works for signed documents too
    → Falls back to documento_visualizar if imprimir_web not found

  client.view_document_sections(id_documento, id_procedimento)
    → Same as above but returns EditorSection list
    → Extracts txaEditor_NNN divs from the print view

  client.read_relatorio(id_documento, id_procedimento)
    → Uses get_editor_sections() → parse_relatorio(html)
    → THIS IS THE IDEAL PATH — but only works for unsigned internal docs

RECOMMENDED FIX for relatorio_read() in operations/reading.py:

  Before falling back to text extraction, try this chain:
    1. client.read_relatorio() — editor path (current, works for unsigned internos)
    2. client.view_document_html() → parse_relatorio(html) — print view path
       (works for signed internos AND should work for documents with HTML representation)
    3. ONLY IF both fail → text fallback (last resort for true binary PDFs)

  The key change in the except block (around line 1754):

    CURRENT (bad):
      doc_payload = _document_read_core(...)  # gets plain text
      relatorio = _fallback_relatorio_from_text(doc_payload["data"]["text"])
      parsing_strategy = "text_fallback"

    PROPOSED (good):
      try:
          html = client.view_document_html(id_documento, id_procedimento_resolved)
          relatorio = parse_relatorio(html)
          parsing_strategy = "structured_html_view"
          extraction_method = "view_document_html"
      except Exception:
          # True last resort — binary PDF with no HTML representation
          doc_payload = _document_read_core(...)
          relatorio = _fallback_relatorio_from_text(doc_payload["data"]["text"])
          parsing_strategy = "text_fallback"

  Same change should apply in _parse_single_relatorio() used by process-report.
